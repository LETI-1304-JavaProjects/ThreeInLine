Результат анализа возможных вариантов реализации поставленной задачи:
Одним из этапов работы над проектом является обеспечения связи между игровыми приложениями-клиентами.
Так как основной целью проекта является не создание полнофункционального сетевого приложения, а освоение основ программирования на Java, то в рамках проекта достаточно будет продемонстрировать работу приложения в упрощенной среде с  примитивным локальным сервером.
Для этого целесообразно разработать интерфейс базовых операций, необходимых для работы и класс, реализующий соответствующие методы этого интерфейса. 
Тогда в будущем, при необходимости, можно будет легко использовать более серьезную реализацию интерфейса без внесения кардинальных изменений в основной код приложения. 
Исходя из этого актуальна задача определения порядка и способа обмена данными между клиентами.

После формирования концепции выполнения приложения могут быть поставлены задачи проектирования логики игры и графического интерфейса пользователя. 
Так как приложение (исходя из поставленной задачи) при любой концепции игры будет содержать данные игрока и игровое поле задача разработки и проектирования графического интерфейса может быть разбита на две части: создание визуализации данных игрока и непосредственно самого игрового поля. Все описанные выше задачи взаимосвязаны.

Задача разработки логики игры влияет на задачи создание визуализации данных игрока и игрового поля и может быть разбита на следующие подзадачи:
1. Формирование поля.
  а. Поиск комбинаций по три.
	б. Поиск хода.
	в. Генерация новых клеток.
2. Проверка хода.
3. Выполнение хода.

При выполнении этих пунктов необходимо учитывать связь с компонентами клиента и графики.
